# Triage Protocol

## 四级分诊

### Level 0 — Emergency（急诊）
Triggered by Class A emergency red flags.
**输出：立即急诊，不继续普通问诊。**

### Level 1 — Urgent（24小时内就医）
Triggered by:
- Class B urgent red flags
- high-risk population
- progressive worsening
**输出：建议当天就医或急诊诊室（非急救）。**

### Level 2 — Routine（门诊）
症状稳定、无危险信号，但需要专业评估。
**输出：建议预约门诊，说明可等待时间范围（如 1-3 天内）。**

### Level 3 — Monitor（观察）
症状轻微、稳定、无危险信号，符合常见良性病因。
**输出：可居家观察，明确恶化标准和升级就医时机。**

## 分诊决策逻辑
1. 先判断是否存在 Class A 红旗 → 有则 Level 0
2. 判断是否存在 Class B 红旗 → 有则 Level 1
3. 判断是否存在 Class C 风险放大器 → 可将 Level 2/3 上调一级
4. 评估人群风险因素（年龄、孕妇、免疫抑制、肿瘤史）
5. 评估症状进展趋势与对日常生活影响
6. 综合给出 Level 1-3 建议

## Triage Overrides

Triage overrides patient psychology, explanation depth, and differential output.
If Level 0 Emergency is triggered, do not use shared-decision framing.
