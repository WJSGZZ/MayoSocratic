# Search Protocol

## 设计原则

本 Skill 采用「内置知识为基线，联网搜索为增强」的分层架构：
- 内置知识：始终可用，覆盖常见主诉的核心红旗和问诊框架
- 联网搜索：部署环境支持时启用，从权威源获取更精准的信息
- 降级处理：搜索不可用时，诚实告知用户信息局限，建议就医

## 触发搜索的条件

以下情况触发联网搜索，不使用内置知识强行填充：
1. 用户主诉的疾病或症状组合不在内置 Illness Scripts 库中
2. 用户描述罕见症状，内置知识不足以支撑鉴别诊断
3. 需要确认某个红旗的最新临床标准
4. 用户询问特定疾病的最新治疗进展

## 权威源优先级

### 西医诊断与红旗

1. mayoclinic.org — 症状页、疾病页、「When to see a doctor」页
2. medlineplus.gov — 美国国立卫生研究院，免费权威
3. nice.org.uk — 英国国家卫生与临床优化研究所，临床指南级别
4. cdc.gov — 传染病、公共卫生相关主诉

### 专科学会（当主诉明确时使用）

- 头痛：americanheadachesociety.org
- 心脏：heart.org（美国心脏协会）
- 糖尿病：diabetes.org
- 癌症筛查：cancer.org

### 中医

- 国家中医药管理局官网（国内标准化证型和治法）
- zysj.com.cn（方剂和证型参考）

## 不应搜索的内容

- 药物具体剂量（超出本 Skill 能力范围，建议咨询药师或医生）
- 处方药使用方案
- 任何论坛、问答平台（知乎、百度知道、丁香园问答区等不是权威源）
- 商业医疗机构的宣传页面

## 搜索失败的处理

搜索不到相关内容时，输出：
「当前信息不足以对你描述的症状给出可靠分析。建议你就医，由医生结合体检和检查评估。」
不要用内置知识强行填充，不要凭空推测。

## 可选能力声明

联网搜索是可选增强能力，依赖部署模型是否支持。
不支持时，退回内置知识，并在输出中注明：
「以下分析基于内置医学知识，未经实时联网验证，仅供参考。」
